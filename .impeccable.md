# Design Context

## Target Audience
Mixed — from beginners to experienced developers. Primary use case is learning authentication/security protocols (currently OpenID Connect).

## Brand Personality
Warm & approachable — friendly but professional, like a well-designed Notion or Linear doc. Not cold or enterprise-y.

## Design Direction
**Warm editorial.** Dark warm sidebar, cream content area. High contrast between nav and content.

## Typography
- **Headings**: Fraunces (optical serif, warm, distinctive) via `.font-heading` CSS class
- **Body**: DM Sans (friendly, readable, not generic)
- **Monospace accents**: Used intentionally for labels, numbers, badges — not as lazy "dev tool" default

## Color System (OKLCH)
- Background: `oklch(97.5% 0.009 78)` — warm cream
- Sidebar: `oklch(13% 0.024 75)` — dark warm near-black
- Accent: `oklch(62% 0.168 52)` — amber/honey
- Secondary: `oklch(54% 0.1 148)` — sage green (used for labs)
- All CSS variables defined in `globals.css` under `:root`

## Key Design Decisions
- Dark sidebar with grouped nav sections (Course / Foundations / Labs / Advanced)
- Amber active states in sidebar (not blue)
- Row-based learning path — NOT identical card grids
- Full tinted background for InfoBox (no border-left + rounded-r)
- CTA block uses solid dark warm background (NOT blue-to-purple gradient)
- Left-aligned hero (not centered)
- Technology badges: monospace, tinted by category (sage for runtime, amber for auth tools)
